## Anti informe vacio

Un reporte de radar debe incluir:

- fuentes consultadas;
- cantidad de candidatos encontrados;
- candidatos descartados y motivo;
- rutas alternativas si una web falla;
- precio/comparable cuando aplique;
- recomendacion accionable o razon verificable para no avanzar.

Si una web no abre, se prueba busqueda dirigida, cache/snippets, Chrome real, otra fuente del mismo mercado o se delega a Pablo antes de cerrar.

## Source recovery obligatorio

Una fuente bloqueada no cierra el radar: abre una recuperacion. Antes de escribir un informe final:

- web inmobiliaria no abre: buscar direccion/zona exacta, portal espejo, snippets/cache y comparables;
- precio inmobiliario faltante: buscar el mismo inmueble o comparables por radio/m2;
- marketplace instrumental bloqueado: buscar marca/modelo en eBay, Mercado Libre, importadores y fabricante;
- modelo ambiguo: buscar ficha/manual/fotos/specs antes de considerarlo oportunidad;
- cero candidatos: registrar queries, fuentes, descartes, universo revisado y proxima corrida.

Frases como "no pude abrir", "no encontre nada" o "no hay oportunidades" quedan bloqueadas salvo que el reporte liste rutas alternativas usadas, evidencia y proxima accion concreta.
